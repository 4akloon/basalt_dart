# diesel

The dialect-agnostic **core** of [diesel_dart](../../README.md) — a type-safe ORM for Dart inspired by
[diesel-rs](https://github.com/diesel-rs/diesel). This package has **no driver dependency**: it defines the
typed schema, expressions, query/write builders, the SQL serializer, and the `Connection` / `SqlDialect`
seams. Concrete backends live in separate packages (e.g. [`diesel_sqlite`](../diesel_sqlite)).

> Part of the diesel_dart workspace. Start with the [root README](../../README.md) and
> [docs](../../docs/getting-started.md).

## What's in here

- **Types** (`SqlType<T>`) — `integer`, `text`, `real`, `boolean`, `blob`, `dateTime`, and `*OrNull`
  variants. See [type-mapping](../../docs/type-mapping.md).
- **Schema** (`schema/table.dart`) — `sealed TableColumn<T, Tbl>` with `ValueColumn` / `PrimaryKey` /
  `Ref<T, Tbl, Target>`; `TableRef`, `QuerySource`, `TableAlias`. Columns are declared `static const` on a
  table marker class so the same object serves the builder and the annotations.
- **Expressions** (`expression/expression.dart`) — `Expression<T, Tbl>` (phantom `Tbl` scope) and the
  `&` / `\|` / `.and` / `.or` combinators.
- **Query builder** (`query/query.dart`) — `from(...).where/orderBy/limit/offset/select/innerJoin/leftJoin`
  → `.map((RowReader r) => …)` → `MappedQuery<R>`. `RowMapper<R>` is the reusable codegen decoder.
- **Writes** (`query/write.dart`) — `insertInto`, `update`, `deleteFrom`, with `TableColumn.set(value)`.
- **Serializer** (`serialize/`) — `QueryBuilder` walks the AST into `(String sql, List<Object?> params)`
  against a `SqlDialect`. Pure and driver-free; this is the unit-testable heart.
- **Execution seam** (`connection.dart`) — the async-first `Connection` interface backends implement.
- **Annotations** (`annotations/`) — `@Queryable`, `@Insertable`, `@AsChangeset`, `@Column`, `@Relation`,
  consumed by [`diesel_codegen`](../diesel_codegen).

## Example

```dart
import 'package:diesel/diesel.dart';

abstract final class Users {
  static const id = PrimaryKey<int, Users>('users', 'id', SqlType.integer);
  static const name = ValueColumn<String, Users>('users', 'name', SqlType.text);
  static const age = ValueColumn<int, Users>('users', 'age', SqlType.integer);
  static const table = TableRef<Users>('users', [id, name, age]);
}

final query = from(Users.table)
    .where((Users.age > 18) & Users.name.like('A%'))
    .orderBy(Users.name.asc())
    .map((r) => (r.get(Users.id), r.get(Users.name)));
// Pass `query` to a Connection's `fetch` (see diesel_sqlite).
```

In real projects the schema is generated by `diesel_dart print-schema` and the decoders by `build_runner`;
see the [derives guide](../../docs/derives.md).
